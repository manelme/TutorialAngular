Resumen

Resumen del curso de OpenWwebinars.

Objetivos

  • Aprender a programar con Angular.

1. Introducción

  • Angular es el framework de Google para el desarrollo de SPA (Single Page Applications). Las SPAs son un tipo de aplicación web en las que la interacción se realiza reescribiendo la página actual en lugar de cargando páginas completas desde el servidor.

  • Angular ofrece una arquitectura que permite desarrollar aplicaciones de gran tamaño, mantenible y que pueda crecer en el futuro. Una aplicación pequeña se puede hacer en Javascript. Una aplicación grande no.

  • Angular está orientado a componentes. Una página está compuesta por varios componentes que se comunican entre sí. Esto nos permite reutilizar componentes en múltiples páginas.

  • Las aplicaciones Angular se desarrollan en TypeScript, aunque admiten otros lenguajes. TypeScript es un superconjunto tipado de JavaScript que se compila a JavaScript plano.

1.1. Versiones de Angular

  • Angular 1 o AngularJS (2010)

  • Angular 2 (2016). Incoroporó gran cantidad de cambios respecto a AngularJS. Los proyectos de AngularJS no son válidos en Angular 2.

  • Angular 4 (Marzo 2017). Incorpora algunas características nuevas respeto a Angular 2. En este caso sí hay compatibilidad.

  • Angular 5 (Noviembre 2017). Nueva versión con nuevas características manteniendo compatibilidad.

Angular y AngularJS son dos proyectos diferentes e incompatibles. Angular comprender las versiones desde Angular 2 en adelante. Actualmente, AngularJS y Angular siguen su desarrollo independiente.

2. Configuración del entorno

Instalaremos lo siguiente:

  • Git.

  • Node.js. En Linux descargar el paquete de Node.js, moverlo a una carpeta /opt/nodejs y crearle un enlace simbólico en /usr/bin/node (ln -s /opt/nodejs/bin/node /usr/bin/node).

  • npm como Gestor de paquetes para JavaScript. npm se distribuye con Node.js.

  • Consola de Angular (Angular CLI) con npm.

    sudo npm install -g @angular/cli

    La opción -g indicar que Angular CLI se instale globalmente por lo que lo podemos usar desde cualquier directorio.
    Angular CLI

    Angular CLI es una herramienta de interfaz de línea de comandos que permite crear proyectos, añadir archivos, y realizar tareas de desarrollo como testing, bundling y deployment.

  1. Desde el directorio de trabajo, crear un proyecto nuevo

    2 Servir el proyecto (serve) y abrir navegador con la aplicación (-o). El proyecto se sirve a través de un servidor web que incorpora Angular y que ofrece live reload, lo que permiteque la aplicación se recargue automáticamente al hacer cambios en los archivos fuente.

4. Organización de un proyecto Angular

Los componentes son los bloques básicos de las aplicaciones Angular. Muestran datos en la pantalla, están atentos a la entrada del usuario, y realizan una acción en función de esa acción.

Figure 2. Organización de una aplicación en componentes

Al crear el proyecto, Angular CLI ha creado el componente principal de la aplicación disponible en src/app/app.component.ts. Al definir un componente, definimos un selector, que es el luego se utilizará en la aplicación para incluir este componente. CLI define como selector para este componente app-root. Este valor se puede modificar.

Example 1. Componente principal de la aplicación Angular
// src/app/component.ts

import { Component } from '@angular/core';

@Component({ (1)
  selector: 'app-root', (2)
  templateUrl: './app.component.html', (3)
  styleUrls: ['./app.component.css'] (4)
})
export class AppComponent { (5)
  title = 'app';
}
1 Objeto metadata que describe las características del componente.
2 Selector del componente. Define una etiqueta HTML personalizada que la aplicación luego usará en los archivos HTML para incluir este componente (ver ejemplo siguiente).
3 Plantilla externa asociado al componente escrita en HTML.
4 Lista de hojas de estilos a aplicar al componente además de la propia de la aplicación (src/styles.css).
5 Exportación de la clase para que puedan ser usadas por otros componentes
Example 2. Referencia a un selector
<!doctype html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <title>PruebaAngular</title>
  <base href="/">

  <meta name="viewport" content="width=device-width, initial-scale=1">
  <link rel="icon" type="image/x-icon" href="favicon.ico">
</head>
<body>
  <app-root></app-root> (1)
</body>
</html>
1 Al incluir el selector app-root se incluirá en este archivo HTML su componente asociado app-component (ver ejemplo anterior).

También es posible incluir el código de la plantilla inline en lugar de en un archivo externo. El código de la plantilla irá entre backsticks o apóstrofes.

// src/app/component.ts

import { Component } from '@angular/core';

@Component({
  selector: 'app-root',
  template: ` (1)
    <h1>
      Welcome to {{ title }}
    </h1>
  `,
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  title = 'app';
}
1 Uso de template para definir la plantilla inline. No olvidar la coma del final si hay más elementos en los metadatos del componente.

4.1. Explorando el código

Podemos cambiar el texto de bienvenida modificando src/app/app.component.html. Ahí vemos que aparece un encabezado <h1> con el texto que hemos visto al abrir la aplicación

<h1>
  Welcome to {{ title }}!
</h1>

Lo cambiaremos por

<h1>
  Bienvenido a {{ title }}!
</h1>
Interpolación

{{title}} es lo que se conoce como sintaxis de interpolación de binding. Se trata de un mecanismo por el podemos asignar un valor a un atributo dentro de un componente. Esta interporlación de binding conecta el componente TypeScript con la plantilla HTML presentando el valor de title en el HTML y toma valor en el momento de renderizar la página. En el ejemplo la asignación del valor se realiza en src/app/app.component.ts. Esto evita manpipular el DOM, ya sea directamente o mediante jQuery.

  <h1>
    Bienvenido a {{ title }}! (1)
  </h1>
1 Presentación (interpolación) del valor en src/app/app.component.html 
  export class AppComponent {
    title = 'app'; (2)
  }
1 Asignación del valor en src/app/app.component.ts

Cambiaremos el valor de title en src/app/app.component.ts por mi primera aplicación Angular.

export class AppComponent {
  title = 'mi primera aplicación Angular';
}

El estilo del componente lo podemos cambiar en src/app/app.component.css. Añadiremos el estilo para <h1>.

h1 {
  color: blue;
  font-size: 250%;
}

Tras estos cambios nuestra aplicación tiene un nuevo aspecto!!

4.2. Archivos de configuración

  • package.json: Fichero de configuración de dependencias.

  • tsconfig.json: Fichero de configuración de Typescript, el lenguaje de interacción con Angular.

  • angular-cli.json: Este fichero sólo está disponible si el proyecto se ha creado con CLI. Establece nombres de carpetas, prefijo de la aplicación y los archivos que se incluyen al crear el proyecto.

  • .editorconfig: Parámetros de configuración para el editor respecto al proyecto (charset, tamaño del espacio de tabulación, …​)

  • .gitignore: Contiene la lista de archivos que no están sujetos a control de versiones del repositorio Git inicializado al crear el proyecto.

Angular CLI también ha incluido un archivo README.md con información de interés, como la creación de componentes (ng generate component component-name), build del proyecto (ng build) y ejecución de pruebas (ng test).

4.3. Estructura de carpetas

  • e2e. Carpeta para pruebas

  • node_modules. Contiene los paquetes instalados.

  • src. Contiene el código del proyecto.

    • app

    • assets. Contiene las imágenes utilizadas en el proyecto.

    • environment. Detalles acerca de los entornos de producción y desarrollo.

4.4. Archivos importantes

  • src/index.html. Es el archivo que se muestra en el navegador. <body> contiene <app-root></app-root>. Este el el selector que se usa en el archivo src/app/app.component.ts, el cual mostrará el archivo src/app/app.component.html.

  • src/app/app.module.ts. Indica a Angular cómo construir la aplicación. También incluye los componentes que forman la aplicación.

  • src/app/app.component.ts es el componente inicial. En nuestro caso asigna el valor app a la variable title y muestra el contenido del template app.component.html aplicándole el estilo app.component.css.

  • src/styles.css. Estilos globales de la aplicación.

  • src/test.ts Punto de entra a los tests unitarios.

5. Creación de componentes

Con Angular CLI también podemos añadir nuevos componentes a la aplicación (ng generate component new-component). Al crear un componente con Angular CLI ocurre lo siguiente:

  • Se modifica el archivo src/app/app.module.ts incluyendo el nuevo componente

  • Se añade una carpeta a src/app con el nombre del nuevo componente. La nueva carpeta incluye los archivos TypeScript, HTML y CSS del nuevo componente.

Podemos usar Angular CLI para crear nuevos componentes

ng generate component heroes

Esta operación hará dos cosas:

  1. Añadirá un nuevo componente (HeroesComponent) a src/app/app.module.ts

    import { AppComponent } from './app.component';

  2. Creará una nueva carpeta src/app/heroes/ con 4 archivos:

    • heroes.component.css

    • heroes.component.html

    • heroes.component.spec.ts

    • heroes.component.ts

En el archivo de la clase del componente encontramos un decorador @Component que especifica los metadatos para el componente. Angular CLI genera tres propiedades de estos metadatos:

  • selector: Selector CSS para el componente. Es el nombre que se usar para hacer referencia al componente desde una plantilla que incluyta este componente.

  • templateUrl: Archivo HTML de la plantilla del componente

  • styleUrls: Array de archivos de estilos CSS propios del componente

Como consideración adicional:

  • La clase del componente se exporta para que otro componente pueda importarla.

  • La clase incluye el constructor constructor() para que lo podamos personalizar.

  • La clase incluye el método ngOnInit() para colocar en él cualquier código de inicialización necesaria para el componente.

5.1. Añadir una propiedad al componente

  1. Crear en la clase /src/app/heroes/heroes.component.ts una nueva propiedad con un valor ficticio (p.e. hero = Superman).

  2. Mostrar la propiedad en la plantilla del componente src/app/heroes/heroes.component.html (p.e. {{hero}})

  3. Incluir el componente en la plantilla de la aplicación src/app/app.component.html. Realmente incluimos el selector usado para el componente definido en src/app/heroes/heroes.component.ts (p.e. <app-heroes></app-heroes>).

5.2. Creación de una clase Hero

Veamos cómo crear una clase para los héroes de forma que puedan tener algo más que un nombre.

  1. Crear la clase src/app/hero.ts. La clase tendrá dos variables de instancia id y name y un constructor para inicializarlas

    Example 4. Clase Hero
    export class Hero {
	id: number;
	name: string;

	constructor(id: number, name: string) {
		this.id = id;
		this.name = name;
	}
}

  2. Instanciar la clase Hero en src/app/heroes.component.ts.

    myHero = new Hero(2, "Batman");

  3. Mostrar los datos del componente añadiendo un binding al componente en src/heroes.component.html

    <p>{{myHero.name}}</p>

5.3. Edición de datos de componentes

Hasta ahora hemos presentado en el template los datos del componente. Aquí veremos cómo actualizarlos creando los que se conoce como un two-way binding. De esta forma los datos van de la propiedad al cuadro de texto y del cuadro de texto a la propiedad.

Haremos el two-way binding con la directiva Angular ngModel. Sin embargo, la directiva ngModel no está disponible de forma predeterminada. Está incluida en el módulo FormsModule que deberemos de importar.

  1. Importar el módulo de formularios en src/app.module.ts, ya que [(ngModel)] no está disponible de forma predeterminada.

    import { FormsModule } from '@angular/forms'; (1)

...

@NgModule({
  ...
  imports: [
    BrowserModule,
    ...
    FormsModule (2)
  ],
  ...
})
    1 Añadir esta línea al principio
    2 Añadir FormsModule a la lista de importaciones

  2. Añadir el cuadro de texto a la plantilla src/app/heroes/heroes.component.html

<input [(ngModel)]="myHero.name">

6. Recorrido de una lista

*ngFor es la directiva de repetición de Angular. Rpetirá el elemento en el que esté definida para cada elemento de la lista. El fragmento siguiente crea tantos elementos <li> como elementos tenga la lista heroes.

<li *ngFor="let hero of heroes">

Para este ejemplo creremos un archivo mock-heroes con un array de héroes para recorrerlo con *ngFor.

7. Control de eventos

Nuestro nuevo objetivo es mostrar los detalles al seleccionar un héroe de la lista.

  1. Añadir en el template src/heroes/heroes.component.html un binding de control de eventos a cada héroe de la lista

    <li *ngFor="let h of heroes" (click)="onSelect(h)"> (1)
  {{h.name}}
</li>
    1 onSelect() es un método de src/heroes/heroes.component.ts que aún no está definido. Este método recibe el objeto Hero sobre el que se ha hecho clic.

  2. Crear el controlador de evento en src/heroes/heroes.component.ts.

    	selectedHero: Hero;

	...

	onSelect(hero: Hero) {
		this.selectedHero = hero;
	}

  3. Actualizar la plantilla src/heroes/heroes.component.html.

    <h2>Detalles de {{selectedHero.name}}</h2>
<label>Nombre: <input [(ngModel)]="selectedHero.name"></label>

8. Uso de condiciones

El problema que teníamos con el planteamiento anterior es que estamos pidiendo mostrar el héroe seleccionado, pero al cargar la página inicialmente no hay héroe seleccionado, por lo que la consola informa de un error. La directiva *ngIf nos ofrece una solución y mostraremos los datos del héroe seleccionado si existe.

Ahora, actualizaremos la plantilla src/heroes/heroes.component.html con el código siguiente:

<div *ngIf="selectedHero">
	<h2>Detalles de {{selectedHero.name}}</h2>
	<label>Nombre: <input [(ngModel)]="selectedHero.name"></label>
</div>

8.1. Aplicación condicional de estilos

También es posible aplicar estilos en función de una condición. El ejemplo lo podemos tener en la aplicación de un estilo al héroe seleccionado. Para ello, comprobaremos si es el héroe seleccionado.

Si dentro del <li> que muestra cada héroe añadimos este código, estamos indicando que aplique la clase CSS selected si el héroe h es el héroe seleccionado.

[class.selected]="hero === selectedHero"

A continuación se muestra cómo quedaría el fragmento entero en el que también hemos añadido el estilo para la lista, y mostramos para cada héroe su id y el nombre.

<ul class = "heroes">
	<li *ngFor="let h of heroes" [class.selected]="h === selectedHero" (click)="onSelect(h)">
		<span class = "badge">{{h.id}}</span>{{h.name}}
	</li>
</ul>

9.2. Interpolación no viva

Permite la presentación de variables de forma estática.

En la interpolación no viva la asignación de valores sólo se realiza una vez y no admite cambios.

 
import { Component } from '@angular/core';

let myVar = "Angular rules!!" (1)

@Component({
  selector: 'app-root',
  template: `
  <h1>
  {{title}}
  </h1>
  <h2>
  ${myVar} (2)
  </h2>
  `,
//  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent {
  title = 'app';
}
1 Declaración de la variable
2 Uso (interpolación) de la variable

Debemos dejar de usar var para pasar a usar let para variables y const para constantes. var produce mucha confusión cuando usamos bloques.
Uso de var Uso de let 
  var saludo = "Hola";

  {
  	var saludo = "Adios";
  }

  console.log(saludo);
 
  let saludo = "Hola";

  {
  	let saludo = "Adios";
  }

  console.log(saludo);

saludo devolvería Adios, el último valor asignado

saludo devolvería Hola porque se ignora la declaración en un ámbito interno.

El uso de var permite la declaración de una variables después de su uso. El intéreprete mueve la declaración antes de su primer uso.

El uso de let no permite declarar una variable después de su uso.
